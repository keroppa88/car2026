CAR2 LOOP DRIVE 仕様まとめ（2026-07-18〜19 / v20260718-car2-59）
実装した内容の最終版。廃止済みの仕様は末尾の記録のみ。

■ キー操作（追加・変更分）
  T   : 全体地図（旧Mから移動）
  M   : 音楽選択モード
  N   : 夜間モード ON/OFF
  B   : 音声切替 3状態（エンジン音 → 車内音 → 無音 → …）
  V   : ボンネットカメラ（一人称視点）
  F   : バックミラー（画面上部の小窓に後方視点）
  Y   : 自動運転
  G   : 表示切替 4状態（全表示 → 操作説明削除 → 曲名削除 → スピード表示削除 → 全表示）
  J/K/L : 曲の10秒巻き戻し / 再生・一時停止 / 10秒早送り（運転中も有効）
  I/O : 前の曲 / 次の曲（運転中も有効）
  ESC : ゲーム一時停止（PAUSE表示＋操作説明パネルを表示）

■ タイトル画面「イニシャルT」
  ・黒バック・オレンジ基調。斜めのオレンジスピードライン2本
    （下側の帯は車両選択ボタンに被らないよう画面最下部寄り）
  ・ロゴ: オレンジ＋太い白縁（5px、Tは7px）の文字を水平に普通に配置。
    傾き・斜体・段差なし。Tだけ大きい
  ・SELECT CAR: Toyota 86 / Volvo 240。クリックで車切替（再読み込み。ゲームは始まらない）
  ・PUSH ANY BUTTON（点滅・オレンジ）/ 最下部に ©keroppa88
  ・何かキーまたはクリックでゲーム開始（開始操作はゲーム側に漏れない）
  ・5秒放置するとデモ画面へ

■ デモ画面（アトラクトモード）
  ・タイトル5秒放置で開始。ゲーム開始後は入らない
  ・走行は自動運転モード（Yと同じ）: 130km/hまで加速して巡航、
    カーブ手前減速、直角に近い急カーブはサイドブレーキでドリフト
  ・音声は車内音（デモ終了時に元の音声モードへ復元）
  ・画面中央に白文字「Demo」を点滅表示
    ヒント:「何かボタンでタイトルへ / スワイプでカメラ」
  ・10秒ごとに昼夜をチェンジ
  ・何かボタン、またはタップでタイトル画面へ戻る（昼に戻して復帰）
  ・スワイプ（ドラッグ）でカメラ移動できる（操作中＋2.5秒はデモの自動カメラより優先）

■ CPU車
  ・道路外に出ない: 走行ラインから横±2.5mに毎フレーム位置を制限。
    ドリフトで膨らんでも壁の外へは出ない（レイキャスト復帰は保険として残存）
  ・ドリフトで曲がる: 車体が先にコーナーへ切れ込み、進行方向が遅れて追従。
    最大約30°の横滑り角。ドリフト中はロール強化＋タイヤスモーク（自車の近くのみ）
  ・テールランプ残像: CPU車のみ（自車は出さない）。
    点ではなく線（前回位置と繋ぐ帯・幅0.5m・横グラデーション）で連続した赤い光の筋。
    自車から320m以内で表示（遠くからも視認可能）。
    残る時間は距離連動: 30m以内=0.375秒（半分・並走時に長すぎないように）〜
    120m以上=0.75秒 を線形補間。0.5m毎・周回ワープ時は繋がない。
    テールランプ本体（車体の発光）は従来のまま

■ 自車の夜間ライト
  ・toyota86: 夜間は toyota86n.vox（ライト拡大版）へ自動切替。
    フロントの黄色部分（純黄255,255,51 と 淡黄255,255,153）が発光。
    RGB比率判定なので同系統の黄色を塗り足せば自動で発光対象になる
  ・volvo240: モデルはそのまま、フロントの白いライトパネル（左右の四角）だけが発光。
    前バンパーの白ナンバープレート・両端の角の小片・リアの白・グリル格子は光らない
    （色＋位置の座標窓で判定）。ヘッド側の小さな光球スプライトは出さない
  ・共通: テールは赤の発光。前方約5mだけ照らす実光源スポットライト
    （光源を車体前端より前に置くため自車の車体・ボンネットには光が当たらない）
  ・自車のテールランプ残像はなし

■ フォグ・空
  ・フォグ開始130m→500m / 空色に溶ける距離480m→1150m。
    近〜中距離の色の霞みを解消しつつ、遠景のぼかし（周回コピーの切れ目隠し）は維持
  ・昼間は青空に雲を表示（ffffx.htmlの雲ドームを流用。速度連動で奥から手前へ流れる。
    CLOUD_CFG の数値で量・大きさ・速さを調整可能）。夜間は非表示

■ 夜間モード（N）
  ・空・フォグはほぼ黒に近い濃紺。環境光・太陽光もかなり暗め
  ・グレー路面のみ昼の1/3程度の明るさを保つ（壁・建物は暗いまま）
  ・道路外の純白マテリアル（FFFFFF）は発光（窓明かり用）。
    SketchUpでビルに純白を追加して map.gltf を書き出せば自動で光る

■ 街灯
  ・車7台分（33.6m）間隔・左右同じ列・壁の少し外側（路面端から約1.5〜3.5m）。
    路面判定で自動配置し、道路上には立てない。出られない場所はスキップ
  ・高さは車3台分（約14.4m）。直立ポールの頂点が道路側へ折れて短い水平アーム、
    先端に地表を向いた平らなランプ
  ・夜は先端が白く発光＋白ハロ（光量4倍）＋地表の光だまり（照射範囲3倍・濃さ1/2）
  ・InstancedMeshで周回コピー分も一括描画（当たり判定なし）

■ 路面判定（白線対応）
  ・白線（白系マテリアルの高さ0付近の面）も走行可能面として扱う
    （白線上で引き戻される＝引っ掛かる問題の修正）
  ・高さ30cm以下の低い段差（白線・ペイントの側面）は壁とみなさない

■ 音声
  ・B: エンジン音 → 車内音 → 無音 の3状態
  ・車内音: 80km/h未満 = sound/drive_on_freeway1.mp3 /
            80km/h以上 = drive_on_freeway2.mp3 をループ（±2km/hの余裕で切替）
  ・どのモードでもドリフト音（タイヤスクリーチ）は鳴る
  ・音量調整: ドリフト音は従来の1.5倍 / エンジン音・燃焼ノイズは20%減（0.8倍）

■ 音楽選択モード（M）
  ・musiclist.txt の「」内ラベルとURLのみ解析（色コード等の他の文言は無視）
  ・開いている間は運転を停止。M / ESC で閉じて運転再開
  ・画面中央に青緑色文字のリスト。↑↓で選択（押しっぱなしは3行ずつ高速閲覧）、
    Enterで再生。マウスも可: クリック=選択 / ダブルクリック=再生
  ・エンジン音など / 音楽 の音量スライダー2本
  ・YouTube: IFrame APIで音声のみ再生。
    ※プレイヤーは400x300のまま画面外(top:-400px)に置く（ffffx.htmlと同じ方式）。
      1pxや透明で隠すと「隠された埋め込み」扱いでエラー150になるため不可
  ・ネットラジオ: HTMLAudioで再生。HLS(.m3u8)は hls.js を自動読み込み
  ・プレイリスト行（URLなし）は今のところ選択しても何も起きない
  ・再生終了で次の曲を自動再生
  ・エラー時の挙動:
      手動選択した曲（Enter/ダブルクリック/I/O） = 飛ばさず「⚠ エラー」表示で停止
      連続再生中（自動送り） = オレンジのトーストを出して次の曲へスキップ
  ・エラー判定はエラーが起きた動画IDと要求IDを照合（ダミーや切替前のエラーで
    選択曲を誤スキップしない）
  ・再生中は画面最下部中央に「」の中身を青緑（21px・細字・くっきり）で表示。
    Gの表示切替（2段階目）で消える
  ・注意: ローカル(http://127.0.0.1)ではレーベル系のYouTube曲がエラー150になる
    ことがある（オリジン制限）。GitHub Pages（https）なら再生できる可能性が高い

■ その他の画面機能
  ・V ボンネットカメラ: ボンネット先端付近から進行方向を見る一人称視点
  ・F バックミラー: 画面上部中央の枠付き小窓に後方視点（ミラー内は自車非表示）
  ・Y 自動運転: 道路中央ラインを追従、巡航130km/h、直角に近い急カーブはドリフト。
    周回の継ぎ目は直進通過後に再同期
  ・ESC 一時停止: PAUSE表示＋操作説明を強制表示。運転キーは受け付けない

■ 実装したが廃止した仕様（記録）
  ・壁の外を覆うモヤ（霧の層） → 廃止
  ・車の光だまり・道路の広い照射 → ボンネットが光って見えるため廃止
    （現在はランプ発光＋自車の短距離実光源のみ）
  ・自車のテールランプ残像（一つ分） → 廃止（残像はCPU車のみ）
  ・YouTubeの映像プレイヤー表示（右上小窓） → 音声のみに変更
  ・地図連結の「3回に1回、180度回転で接続」 → 実装後に撤廃（常に同じ向きの単純ループ）
  ・J/K 2連続早押しでの前曲/次曲 → 廃止（I/Oキーに変更）
  ・タイトルの「TOUGE DRIFT RACING」キッカー・文字の傾き/波打ち配置 → 削除
    （現在は水平・無傾斜のオレンジ白縁文字）
